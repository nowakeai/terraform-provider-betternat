# terraform-provider-betternat v0.1.0 Release Notes

Date: 2026-06-24

## Status

`v0.1.0` is the first GA Terraform/OpenTofu provider release for BetterNAT.
Use it with BetterNAT runtime `v0.1.0`:

```hcl
terraform {
  required_providers {
    betternat = {
      source  = "nowakeai/betternat"
      version = "= 0.1.0"
    }
  }
}

resource "betternat_gateway" "egress" {
  name   = "prod-egress-a"
  region = "us-west-2"
  vpc_id = aws_vpc.main.id

  public_subnet_ids = {
    "us-west-2a" = aws_subnet.public_a.id
  }

  private_route_table_ids = {
    "us-west-2a" = [aws_route_table.private_a.id]
  }

  private_cidrs = [aws_vpc.main.cidr_block]

  ami_id            = data.aws_ami.ubuntu_arm64.id
  instance_type     = "t4g.small"
  betternat_version = "v0.1.0"
  stable_egress_ip  = true
}
```

Terraform Registry is the primary install path. The provider is also published
through the OpenTofu Registry with the same `nowakeai/betternat` source address.

## What Changed Since the Alpha Line

- Added built-in BetterNAT runtime `v0.1.0` artifact URLs and SHA256 checksums
  for normal `bootstrap_mode = "cloud_init"` installs.
- Updated the Registry overview, resource documentation, and examples to use
  provider `0.1.0` and runtime `v0.1.0`.
- Kept the Terraform schema compatible with the late alpha line.
- Kept the default non-AMI bootstrap path: users provide a Linux AMI and the
  provider installs BetterNAT release artifacts at first boot.
- Documented `associate_public_ip_address` behavior for environments that need
  to override per-node public IPv4 assignment.

## Known Limitations

- AWS only.
- One AZ per `betternat_gateway` resource. Create one resource per AZ for
  multi-AZ topologies.
- No NAT Gateway equivalent SLA.
- Existing connections are not preserved across failover; new flows converge
  after route/EIP ownership moves.
- No public BetterNAT AMI is published. The supported default path is
  `bootstrap_mode = "cloud_init"` with an explicit Linux AMI.
- `ami_channel` resolution is reserved for future AMI-based installs.
- Updating `betternat_version` changes launch-template/bootstrap metadata; use
  an instance refresh, replacement, or blue/green workflow to prove running
  gateway nodes moved to the new binaries.

## Artifact Integrity

Provider release assets include `SHA256SUMS`, a detached GPG signature, and a
registry manifest. Verify downloaded assets before using filesystem mirrors or
manual installation.
