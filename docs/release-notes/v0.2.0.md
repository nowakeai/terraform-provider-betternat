# v0.2.0

## Terraform Surface Reset

`v0.1.1` was published but not promoted and has no known external users. This
release intentionally resets the Terraform surface before broader public
promotion.

Use `betternat_aws_gateway` for AWS gateway installs. The old
`betternat_gateway` resource is removed from the primary provider surface.

## What Changed

- Added `betternat_aws_gateway`.
- Added alpha `betternat_gcp_gateway` for GCP HA validation with Firestore
  coordination, tagged route ownership, LoxiLB bootstrap, MIG capacity repair,
  and optional stable public identity.
- Added `betternat_runtime_artifacts`.
- Added `betternat_aws_gateway_status`.
- Added alpha `betternat_gcp_gateway_status` for read-only GCP Compute route
  and instance status.
- Updated Registry docs and examples to use cloud-specific resource naming.

## Who Should Use This

Use this release for new AWS BetterNAT Terraform work after the surface reset.
Do not use it as an in-place upgrade from `v0.1.1`; create a replacement plan
and review route ownership carefully.

User docs:

- AWS Quick Start: <https://github.com/nowakeai/betternat/blob/main/docs/user/getting-started/QUICK_START.md>
- GCP Quick Start: <https://github.com/nowakeai/betternat/blob/main/docs/user/getting-started/GCP_QUICK_START.md>

## Validation

Release validation must include:

- `go test ./...`
- local provider build,
- Terraform dev-override validation for provider examples,
- AWS disposable VPC smoke before promotion.
- GCP disposable module smoke with Firestore HA, private-client egress,
  handover, and cleanup evidence before promoting the GCP alpha docs.

## Compatibility

This release is intentionally breaking relative to unpromoted `v0.1.1`.
Normal SemVer compatibility discipline resumes after this reset.
