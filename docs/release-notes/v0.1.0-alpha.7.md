# terraform-provider-betternat v0.1.0-alpha.7 Release Notes

Date: 2026-06-24

## Status

`v0.1.0-alpha.7` is the current alpha Terraform provider release for BetterNAT.

Use it with the current BetterNAT runtime artifact version:

```hcl
betternat_version = "v0.1.0-alpha.2"
```

This release is still an alpha technical preview. Test in a disposable or
non-critical AWS environment before using it with real private route tables.

## What Changed Since v0.1.0-alpha.6

- Added `bootstrap_mode` to `betternat_gateway`.
- Added `associate_public_ip_address` to `betternat_gateway`.
- Default `cloud_init` gateway nodes now get ordinary auto-assigned public IPv4
  addresses for first-boot package, Docker image, BetterNAT artifact, SSM, and
  AWS control-plane reachability.
- `bootstrap_mode = "prebaked_ami"` is available for private or future
  BetterNAT AMIs. In stable-EIP mode, the provider-derived behavior disables
  per-node auto-assigned public IPv4 because the runtime is already present.
- The shared EIP remains the intended stable private-workload egress identity
  when `stable_egress_ip=true`.
- The provider remains compatible with BetterNAT runtime
  `v0.1.0-alpha.2`.

## Example

```hcl
terraform {
  required_providers {
    betternat = {
      source  = "nowakeai/betternat"
      version = "= 0.1.0-alpha.7"
    }
  }
}

resource "betternat_gateway" "egress" {
  name   = "prod-egress-a"
  region = "us-west-2"
  vpc_id = aws_vpc.main.id

  public_subnet_ids = {
    "us-west-2a" = aws_subnet.public_a.id
  }

  private_route_table_ids = {
    "us-west-2a" = [aws_route_table.private_a.id]
  }

  private_cidrs = [aws_vpc.main.cidr_block]

  ami_id            = data.aws_ami.al2023_arm64.id
  betternat_version = "v0.1.0-alpha.2"
  stable_egress_ip  = true

  # Optional advanced override. Leave unset for the provider-derived default.
  # associate_public_ip_address = true
}
```

## Known Limitations

- AWS only.
- Single-AZ HA group only.
- No NAT Gateway equivalent SLA.
- No active connection preservation.
- No public BetterNAT AMI is published.
- `ami_channel` resolution is not implemented.
- OpenTofu Registry propagation can lag Terraform Registry propagation. During
  this validation window, Terraform Registry exposed `0.1.0-alpha.7` while the
  OpenTofu Registry still reported `0.1.0-alpha.6`.
- In stable-EIP mode, gateway nodes can also have ordinary per-node public IPv4
  addresses for bootstrap and management reachability. During the clean AWS
  validation, proactive handover recorded `1` curl timeout and `2` transient
  samples from the standby node's ordinary public IPv4 before traffic returned
  to the shared EIP. Strict "every successful handover sample returns only the
  shared EIP" semantics likely require a future secondary private IP or
  secondary ENI egress identity.
