# terraform-provider-betternat v0.1.0-alpha.9 Release Notes

Date: 2026-06-24

## Status

`v0.1.0-alpha.9` is an alpha Terraform provider release for BetterNAT. It was
published after BetterNAT runtime `v0.1.0-alpha.8`, but its built-in
`betternat_version` artifact manifest still accepts the same runtime versions
as provider `v0.1.0-alpha.8`.

For normal alpha installs, keep using:

```hcl
betternat_version = "v0.1.0-alpha.6"
```

Runtime `v0.1.0-alpha.8` can still be tested with explicit
`agent_binary_url`, `agent_binary_sha256`, `cli_binary_url`, and
`cli_binary_sha256` overrides. Do not treat `v0.1.0-alpha.9` as a clean
artifact-manifest release for `betternat_version = "v0.1.0-alpha.8"`.

This release is still an alpha technical preview. Test in a disposable or
non-critical AWS environment before using it with real private route tables.

## What Changed Since v0.1.0-alpha.8

- Updated the provider dependency to BetterNAT main module
  `v0.1.0-alpha.8`, which includes handover route replacement retry and
  ambiguous-error verification behavior.
- No Terraform schema change is intended in this release.
- The normal `betternat_version` path still accepts the alpha6 runtime artifact
  set. Use explicit artifact overrides when validating runtime alpha8.

## Example

```hcl
terraform {
  required_providers {
    betternat = {
      source  = "nowakeai/betternat"
      version = "= 0.1.0-alpha.9"
    }
  }
}

resource "betternat_gateway" "egress" {
  name   = "prod-egress-a"
  region = "us-west-2"
  vpc_id = aws_vpc.main.id

  public_subnet_ids = {
    "us-west-2a" = aws_subnet.public_a.id
  }

  private_route_table_ids = {
    "us-west-2a" = [aws_route_table.private_a.id]
  }

  private_cidrs = [aws_vpc.main.cidr_block]

  ami_id            = data.aws_ami.ubuntu_arm64.id
  betternat_version = "v0.1.0-alpha.6"
  stable_egress_ip  = true
}
```

## Validation Before Publication

- BetterNAT runtime `v0.1.0-alpha.8` release workflow: passed.
- BetterNAT runtime `v0.1.0-alpha.8` release asset checksum verification:
  passed.
- Split provider `go test ./...`: passed locally before release.
- Split provider build: passed locally before release.

Terraform Registry install is validated for this provider release. Runtime
`v0.1.0-alpha.8` ASG lifecycle validation should use explicit artifact
overrides unless a later provider release adds a clean `betternat_version`
artifact-manifest entry.

## Known Limitations

- AWS only.
- Single-AZ HA group only.
- No NAT Gateway equivalent SLA.
- No active connection preservation.
- No public BetterNAT AMI is published.
- `ami_channel` resolution is not implemented.
- Updating `betternat_version` changes the launch template and bootstrap
  metadata, but existing gateway nodes must still be replaced or refreshed to
  prove the running binaries changed.
