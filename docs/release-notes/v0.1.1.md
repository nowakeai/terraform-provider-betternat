# terraform-provider-betternat v0.1.1 Release Notes

Date: 2026-06-25

## Status

`v0.1.1` is a patch release for the Terraform/OpenTofu provider. Use it with
BetterNAT runtime `v0.1.0`:

```hcl
terraform {
  required_providers {
    betternat = {
      source  = "nowakeai/betternat"
      version = "= 0.1.1"
    }
  }
}

resource "betternat_gateway" "egress" {
  betternat_version = "v0.1.0"
  # ...existing gateway configuration...
}
```

## What Changed Since v0.1.0

- Hardened destroy cleanup for provider-managed security groups.
- Before deleting the gateway node security group, the provider now checks
  `DescribeNetworkInterfaces` with `group-id` to verify whether AWS still has
  dependent ENIs attached.
- `DependencyViolation` from `DeleteSecurityGroup` is retried after another
  dependency inspection, and timeout errors now include the last visible ENI
  dependencies to make cleanup failures diagnosable.

This is not a blind sleep increase. The destroy path waits on the concrete EC2
dependency that blocks security group deletion.

## Compatibility

- No Terraform schema change.
- No BetterNAT runtime release change.
- No replacement is expected for unchanged gateway configuration.
